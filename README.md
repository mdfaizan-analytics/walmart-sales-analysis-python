# Walmart Retail Sales Performance Analysis

An end-to-end exploratory data analysis (EDA) project built with Python and Jupyter Notebooks to clean, aggregate, and visualize weekly sales revenue across multiple Walmart retail store locations.

## 📌 Project Overview
This project processes historical corporate retail data from Kaggle containing over 6,400 transaction checkpoints. The core objective is to analyze regional performance metrics, restructure data views to handle scale (bypassing scientific notation formats), and extract business intelligence on operational performance across physical locations.

## 📊 Business Insights Extracted
* **Total Network Sales Revenue:** ~$6.74 Billion across all recorded data logs.
* **Average Store Weekly Revenue:** $1,046,964.88
* **Top Performing Location:** **Store 20** yielded the highest cumulative sales volume tracking at **$301,397,792.46**.
* **Lowest Performing Location:** **Store 33** yielded the lowest cumulative sales volume tracking at **$37,160,221.96**.

## 🛠️ Tech Stack & Environment
* **Language:** Python 3
* **Environment:** Jupyter Notebook Engine
* **Libraries Used:**
  * **Pandas:** Grouping operations, multi-index aggregation, and floating-point format override rules.
  * **NumPy:** Mathematical data type tracking.
  * **Matplotlib:** Core plotting engine for automated bar distribution visualizations.
